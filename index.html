Доплаты диспетчеров по сделкам

      

        
Доплата считается только с переданных партнёрам сделок — собственные сделки (партнёр «—») доплату не дают вообще, а идут только в выручку компании. Доля компании зависит от партнёра:

        

          

            Партнёр
            Доля компании
            Доплата диспетчеру
            Итого от сделки
          

          
            
ТП20% от суммы сделки10% от доли компании2% от сделки

            
Выездной шиномонтаж15% от суммы сделки10% от доли компании1.5% от сделки

            
Лоджик / А24 / КарТакси / Подрядный эвакуаторСумма сделки − Стоимость у партнёра
или КВ партнёра, если заполнено (приоритет у КВ)		10% от доли компании10% от разницы или КВ

          
        

        
Дополнительные правила:

        
    
          
  • Сделки без заполненного поля «Водитель / Механик» в расчёт не попадают.
    • 
          
  • Если заполнено поле «Ответственный за передачу заявки» — доплата идёт ему, а не ответственному за сделку (в таблице помечается значком ↗).
    • 
          
  • ID партнёра «1075» (прочерк «—») означает собственную сделку — не даёт доплаты, идёт в выручку компании полностью.
    • 
        

      

    


    

      
Выручка компании

      

        
Выручка компании = сумма по всем сделкам, где:

        
    
          
  • Собственные сделки (партнёр «—») — полная сумма сделки (поле OPPORTUNITY).
    • 
          
  • Партнёрские сделки — только доля компании (см. таблицу выше), не вся сумма заказа.
    •

Расписание смен

      

        
Правила автогенерации (кнопка «⚡ Генерировать»):

        
    
          
  • Юля — 5/2, пн-пт смена 8-16, сб-вс выходной
    • 
          
  • Зубцов — 5/2, пн-пт смена 7-15, сб-вс выходной
    • 
          
  • Никита — утро/вечер, может работать день, не работает ночь
    • 
          
  • Вова — день, может работать двойную утро+вечер, не работает ночь
    • 
          
  • Вероника — может работать любую смену
    • 
          
  • Стас — приоритет ночные смены, может работать день, не работает утро
    • 
          
  • Каждый день должно быть покрытие: утро (8-16), вечер (16-00 либо двойная утро+вечер), ночь (00-08 либо двойная вечер+ночь), день (9-21)
    • 
          
  • При создании нового месяца расписание копируется из предыдущего со смещением по дням недели (понедельник остаётся понедельником)
    • 
        

        
Часы смен: 8-16 / 16-00 / 00-08 / 7-15 = 8ч; 9-21 = 12ч; двойные смены = 16ч.

      

    


    

      
Конверсия диспетчерской

      

        
Конверсия = успешные сделки (включая «Сбор обратной связи») / все сделки, кроме исключённых.

        
Всегда исключается из конверсии:

        
    
          
  • Стадия «Эквивалент»
    • 
          
  • Причина отказа «Негабарит» (и в Приценке, и в Принятом заказе)
    • 
        

        
Исключается только для Клубных клиентов:

        
    
          
  • Причина «Машина завелась сама» — для Клубных это хорошо для компании (подписка не требует выезда), для Разовых/Корп/Обычных/Самонайденных считается отказом
    • 
        

        
«Приценка» (услуга) и «Сбор обратной связи» (стадия):

        
    
          
  • Приценка считается в знаменателе конверсии (общее кол-во), но никогда не попадает в успешные — приценка всегда отказ, даже если по ней потом что-то оформили отдельной сделкой
    • 
          
  • «Сбор обратной связи» (FINAL_INVOICE) приравнена к «Сделка успешна» (WON) — обе считаются выполненными
    • 
        

        
Можно дополнительно исключить любую конкретную причину отказа кнопкой «−» в таблице причин — действует мгновенно без перезагрузки данных.

      

    


    

      
Конверсия отдела продаж

      

        
Воронка «Продажа новых карт» (CATEGORY_ID=0). Логика конверсии такая же, как для диспетчерской: успешные («Сделка успешна») / все сделки. Менеджеры: Бородина Елена, Блинков Андрей, Якунина София.

      

    


    

      
Оперативный отчёт

      

        
    
          
  • «Выполнено» считается по стадиям «Сделка успешна» и «Сбор обратной связи» с реальным временем окончания обслуживания (не по CLOSEDATE — у незавершённых стадий это поле может быть пустым)
    • 
          
  • Период по умолчанию — с 08:00 текущего дня, можно менять время «От/До»
    • 
          
  • Партнёрские сделки по умолчанию исключены из отчёта; галочка «Включить партнёров (КВ)» добавляет их с учётом только доли компании, а не полной суммы заказа
    • 
          
  • Разбивка по категориям: Разовые+Корп (включая Самонайденный/Обычный) и отдельно ЧК (Клубные)
    • 
          
  • Сравнение «Вчера в это время» — те же стадии и временной диапазон, но за вчерашний день
    • 
        

      

    


    

      
Корпоративные клиенты

      

        
    
          
  • Сумма заказа: сначала берётся поле «Стоимость корпоративного заказа», если пусто — берётся сумма сделки (OPPORTUNITY), если и оно пусто — парсится первое число из «Комментариев»
    • 
          
  • Две отдельные таблицы — ТП и Эвакуация — с разбивкой по дням месяца для каждого клиента
    • 
          
  • Третья секция — заявки от мастеров-приёмщиков (источник обращения ДЦ = «Мастер-приемщик»), компания берётся из поля «Источник обращения МП»
    •
